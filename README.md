## First 5 minutes — open `STRATEGY_FINDINGS.md`

It has the bar your agent has to beat:

```
On the official Prophet Arena Subset-1200 benchmark:
  Aggregate:  −$51 (best fixed baseline: noisy + tight_band)
  Sports:     −$2  (essentially flat — the bid-ask spread is the only adversary)
```

If our agent does better than this on `subset_1200`, we have real edge.

## Next 10 minutes — run the harness

```bash
cd prep
pip install -r requirements.txt
cat data/summary.md                                              # what we have
python scripts/strategy_comparison.py                            # run on the authoritative benchmark
python scripts/strategy_comparison.py --source local             # run on our self-polled live data
python scripts/strategy_comparison.py --category Sports          # the meaningful category
```

To plug in a new forecaster or strategy:

- Forecasters live at `prep/src/prep/baselines/<name>.py` (must export `predict(event) -> dict`)
- Strategies live in `prep/src/prep/trade.py` — add a function with the
  signature `(p_yes, yes_ask, no_ask) -> BetSignal | None`

## What we know that's non-obvious

These came from reading the paper + running our own backtests. See `STRATEGY_FINDINGS.md` and `PAPER_NOTES.md` for details.

1. **Trading is Kalshi-only.** The hackathon's `ai-prophet` codebase has a Kalshi adapter and nothing else. Polymarket / Manifold / Metaculus data is not eligible.
2. **You don't choose markets — organizers do.** The agent receives a `candidate_set` per tick.
3. **Best forecaster ≠ best trader.** GPT-5 wins Brier but Claude Opus 4.1 wins market returns. Use Claude Opus 4.1 or GPT-4o.
4. **Sports is the most efficient market** — hardest to beat. Politics / Economics / World have more LLM-exploitable inefficiency.
5. **All 24 evaluated models have negative Sharpe.** The realistic goal is *lose less than other teams*, not make money.
6. **`tight_band` (spread filter 0.95–1.02) is the universal winner** across all forecasters and categories tested.
7. **`kelly_lite` is the universal worst** — Kelly over-sizes when probabilities are noisy.
8. **Bi-direction prompting is a free win:** ask P(YES) and P(NO), then average.
